# Analytical SQL Queries in Restaurant Dashboard

This document lists the core analytical SQL queries generated by `AnalyticsService.php` to power the dashboard charts and metrics. While the application uses Laravel's Eloquent ORM and Query Builder, the underlying SQL executions are responsible for aggregating the database entries.

## 1. Daily Order Counts
**Purpose:** Calculates the total number of orders placed on each distinct day over a given date range for a specific restaurant. This data is fed into the `DailyOrdersChart`.

**Underlying SQL:**
```sql
SELECT 
    DATE(ordered_at) as date, 
    COUNT(*) as count
FROM orders
WHERE restaurant_id = ? 
  AND ordered_at BETWEEN ? AND ?
GROUP BY DATE(ordered_at)
ORDER BY DATE(ordered_at) ASC;
```

---

## 2. Daily Revenue
**Purpose:** Calculates the total revenue generated on each day over a given date range for a specific restaurant by summing the `order_amount`. This data is fed into the `DailyRevenueChart`.

**Underlying SQL:**
```sql
SELECT 
    DATE(ordered_at) as date, 
    SUM(order_amount) as revenue
FROM orders
WHERE restaurant_id = ? 
  AND ordered_at BETWEEN ? AND ?
GROUP BY DATE(ordered_at)
ORDER BY DATE(ordered_at) ASC;
```

---

## 3. Average Order Value (AOV)
**Purpose:** Calculates the average order value across an entire date range for a specific restaurant.

**Underlying SQL:**
```sql
SELECT 
    AVG(order_amount) as aggregate
FROM orders
WHERE restaurant_id = ? 
  AND ordered_at BETWEEN ? AND ?;
```

---

## 4. Peak Hour Per Day
**Purpose:** Determines the busiest hour of the day (highest order count) for each day in a date range for a particular restaurant. Used in the `PeakHourChart`.

**How it works:** First, SQL is used to aggregate the count of orders for every distinct day-hour combination. Then, the application logic (in PHP) loops through the grouped results to pick the highest `order_count` for each distinct `date`.

**Underlying SQL:**
```sql
SELECT 
    DATE(ordered_at) as date, 
    HOUR(ordered_at) as hour, 
    COUNT(*) as order_count
FROM orders
WHERE restaurant_id = ? 
  AND ordered_at BETWEEN ? AND ?
GROUP BY DATE(ordered_at), HOUR(ordered_at)
ORDER BY DATE(ordered_at) ASC, order_count DESC;
```

---

## 5. Top Restaurants by Revenue
**Purpose:** Identifies the top performing restaurants platform-wide by summing their total revenue and total orders within a specific date timeframe. This is typically used for leaderboard features or admin overarching stats.

**Underlying SQL:**
```sql
SELECT 
    restaurants.id, 
    restaurants.name, 
    restaurants.cuisine, 
    restaurants.location, 
    restaurants.rating, 
    SUM(orders.order_amount) as total_revenue, 
    COUNT(orders.id) as total_orders
FROM restaurants
INNER JOIN orders ON restaurants.id = orders.restaurant_id
WHERE orders.ordered_at BETWEEN ? AND ?
GROUP BY 
    restaurants.id, 
    restaurants.name, 
    restaurants.cuisine, 
    restaurants.location, 
    restaurants.rating
ORDER BY total_revenue DESC
LIMIT 3;
```

---

## 6. Granular Orders Filter
**Purpose:** Fetches a paginated, filtered list of orders for detailed review. It allows filtering across multi-dimensions, including the hour of the day using `HOUR()`.

**Underlying SQL (Example with some filters applied):**
```sql
SELECT * 
FROM orders
WHERE restaurant_id = ?
  AND ordered_at >= ?             -- Start date
  AND ordered_at <= ?             -- End date
  AND order_amount >= ?           -- Min amount filter
  AND HOUR(ordered_at) >= ?       -- Hour range from
  AND HOUR(ordered_at) <= ?       -- Hour range to
ORDER BY ordered_at DESC
LIMIT 15 OFFSET 0;
```
